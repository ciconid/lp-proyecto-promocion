# Subinciso 3

## Dureza del sistema de tipos

### Definición de dureza
La dureza del sistema de tipos se refiere al grado de rigidez con el que el lenguaje impone las reglas de tipado y detecta errores en tiempo de compilación. Un lenguaje duro rechaza más programas ambigüos o potencialmente inseguros, mientras que uno blando permite mayor flexibilidad en el código.

### Scala en términos de dureza
Scala tiene un sistema de tipos estático y fuerte:

- Verifica tipos en compilación.
- Soporta inferencia de tipos, pero mantiene comprobaciones estrictas.
- Permite polimorfismo paramétrico y tipos algebraicos con `sealed trait` y `case class`.
- Incorpora `Option` para manejar ausencia de valor de forma segura.
- Admite conversiones implícitas, lo que puede disminuir la percepción de dureza si se usan en exceso.

Ejemplo de tipado estático:

```scala
def suma(x: Int, y: Int): Int = x + y

val total = suma(2, 3)
```

### Comparación con Rust
Rust es un lenguaje de tipos aún más duro que Scala:

- Su tipado es estático, fuerte y sin null implícito.
- Añade ownership, borrowing y lifetimes, que imponen restricciones adicionales.
- Detecta errores de memoria y de concurrencia en compilación.
- Requiere anotaciones de tipo más explícitas en estructuras complejas.

Ejemplo de borrow checker de Rust:

```rust
fn imprimir(texto: &String) {
    println!("{}", texto);
}

fn main() {
    let s = String::from("hola");
    imprimir(&s);
}
```

### Comparación frente a otros lenguajes
Tomando como referencia un lenguaje dinámico como Python, Scala y Rust son notablemente más duros:

- Python realiza la mayoría de los chequeos en tiempo de ejecución.
- Scala detecta muchos errores antes de ejecutar el programa.
- Rust extiende este chequeo a la seguridad de memoria y la gestión de recursos.

### Conclusión

Scala ofrece un equilibrio entre dureza y expresividad: su sistema de tipos es rígido pero permite una sintaxis compacta gracias a la inferencia. Al comparar con Rust, se observa que Rust es más duro, especialmente en los aspectos de seguridad y administración de memoria. Esta dureza adicional lo vuelve más seguro, pero también más exigente para el programador.

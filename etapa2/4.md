# Subinciso 4

## Documento de análisis con ejemplos ilustrativos

Este documento resume los puntos clave del análisis del lenguaje seleccionado y presenta ejemplos concretos que ayudan a comprender los conceptos.

### 1. Incorporación de unidades y pasaje de parámetros
Scala organiza el código en paquetes, objetos, clases y traits. Un archivo puede contener varias definiciones, pero la unidad lógica de diseño se construye con paquetes y objetos.

Ejemplo de módulo simple:

```scala
package recopilador

trait Lector {
  def leer(ruta: String): String
}

object LectorArchivo extends Lector {
  def leer(ruta: String): String = scala.io.Source.fromFile(ruta).mkString
}
```

El pasaje de parámetros en Scala es flexible:

```scala
def ejecutar(archivo: String, procesar: String => String): String = {
  val texto = LectorArchivo.leer(archivo)
  procesar(texto)
}
```

Este diseño separa lógica de entrada/salida de procesamiento, tal como piden los requisitos de implementación.

### 2. Mecanismos de concurrencia
Scala puede usar `Future` para tareas concurrentes sobre la JVM:

```scala
import scala.concurrent.{Future, ExecutionContext}
import ExecutionContext.Implicits.global

val resumenFuturo = Future {
  procesarDocumentos("docs/")
}
```

También se pueden usar actores para modelos de mensajería, lo que facilita el diseño de sistemas concurrentes más complejos.

### 3. Constructores de estilo funcional
Scala promueve la inmutabilidad y las funciones como valores.

Ejemplo con colecciones:

```scala
val oraciones = List("a", "b", "c")
val tokens = oraciones.flatMap(_.split(" "))
val importantes = tokens.filter(_.length > 3)
```

El uso de `case class` y `pattern matching` permite trabajar con estructuras de datos de forma funcional:

```scala
case class Oracion(texto: String, puntuacion: Double)

oraciones match {
  case head :: tail => println(head.texto)
  case Nil => println("Sin oraciones")
}
```

### 4. Relación entre estilo funcional y OO
Scala integra ambos paradigmas. Un mismo programa puede combinar objetos, herencia y funciones de primera clase.

Ejemplo:

```scala
trait Calculador {
  def calcular(tokens: List[String]): Double
}

class Puntuador extends Calculador {
  def calcular(tokens: List[String]): Double = tokens.map(_.length).sum.toDouble
}
```

### 5. Sistema de tipos y chequeos
Scala tiene tipado estático con inferencia, lo que permite escribir código claro y seguro.

Ejemplo de `Option` para evitar `null`:

```scala
def buscarToken(token: String, lista: List[String]): Option[String] =
  lista.find(_ == token)

buscarToken("TF-IDF", List("TF", "IDF")) match {
  case Some(valor) => println(valor)
  case None => println("No encontrado")
}
```

### Conclusión general

El análisis muestra que Scala es un lenguaje capaz de soportar un diseño modular y funcional, con un sistema de tipos estático y un buen soporte de concurrencia. En comparación con Rust, Scala privilegia la expresividad y la productividad, mientras que Rust ofrece un sistema de tipos más duro y un control más directo sobre la memoria. Esta síntesis sirve como base para el desarrollo de la aplicación de resumen extractivo y para el informe final.

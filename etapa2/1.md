# Subinciso 1

## Lenguaje seleccionado: Scala

### Incorporación de unidades
Scala organiza el código en paquetes (`package`), objetos (`object`), clases (`class`) y traits (`trait`). Cada archivo fuente puede contener varias definiciones, pero la unidad lógica de compilación es el paquete y el objeto/clase principal.

- `package` agrupa nombres y evita colisiones.
- `object` define un singleton que puede contener métodos y valores estáticos.
- `trait` permite definir interfaces y mixins reutilizables.

Fuentes:
- Paquetes e imports: https://docs.scala-lang.org/tour/packages-and-imports.html
- Objetos singulares: https://docs.scala-lang.org/tour/singleton-objects.html
- Traits: https://docs.scala-lang.org/tour/traits.html

Ejemplo:

```scala
package app.text

trait Procesador {
  def procesar(texto: String): String
}

object ResumenApp extends App {
  println("Resumen iniciado")
}
```

### Pasaje de parámetros
Scala pasa argumentos por valor por defecto, pero también admite pasaje por nombre (`=>`) y valores por defecto.

- Por valor: `def f(x: Int): Int`.
- Por nombre: `def repetir(expr: => Int): Int`.
- Parámetros con nombre y valores por defecto facilitan la legibilidad.

Fuentes:
- Parámetros por nombre: https://docs.scala-lang.org/tour/by-name-parameters.html
- Argumentos nombrados: https://docs.scala-lang.org/tour/named-arguments.html
- Valores por defecto: https://docs.scala-lang.org/tour/default-parameter-values.html

Ejemplo:

```scala
def medir[T](expr: => T): T = {
  println("Iniciando evaluación")
  val resultado = expr
  println("Evaluación finalizada")
  resultado
}

val n = medir { 1 + 2 }
```

### Mecanismos de concurrencia
Scala aprovecha la plataforma JVM y ofrece varias abstracciones:

- `Thread` y `Runnable` para concurrencia clásica.
- `Future` y `Promise` en `scala.concurrent` para programación asíncrona.
- `ExecutionContext` para ejecutar tareas concurrentes.
- Actores (por ejemplo, Akka) para modelos de mensajería.

Fuentes:
- Futures y Promises: https://docs.scala-lang.org/overviews/core/futures.html
- Ejecución concurrente con ExecutionContext: https://docs.scala-lang.org/overviews/core/futures.html#execution-context
- Actores con Akka: https://doc.akka.io/docs/akka/current/typed/actors.html

Ejemplo básico con `Future`:

```scala
import scala.concurrent.{Future, ExecutionContext}
import ExecutionContext.Implicits.global

val tarea = Future {
  Thread.sleep(500)
  "Resultado"
}

tarea.foreach(println)
```

### Constructores de estilo funcional
Scala incorpora construcciones funcionales centrales:

- Inmutabilidad con `val`.
- Funciones de primera clase y lambdas.
- Colecciones inmutables y operaciones `map`, `filter`, `flatMap`.
- `case class` y `pattern matching` para datos algebraicos.

Fuentes:
- Funciones y lambdas: https://docs.scala-lang.org/tour/higher-order-functions.html
- Colecciones: https://docs.scala-lang.org/overviews/collections/introduction.html
- Case classes: https://docs.scala-lang.org/tour/case-classes.html
- Pattern matching: https://docs.scala-lang.org/tour/pattern-matching.html

Ejemplo:

```scala
case class Documento(titulo: String, contenido: String)

val docs = List(
  Documento("A", "..."),
  Documento("B", "...")
)

val titulos = docs.map(_.titulo)
```

### Relación con el paradigma orientado a objetos
Scala es un lenguaje híbrido: todo es objeto y cada operación se expresa como llamada a método.

- Las clases y objetos se combinan con traits.
- La herencia simple se extiende con traits como mixins.
- No existen tipos primitivos independientes; incluso `Int` y `String` son objetos.

Fuentes:
- Orientación a objetos en Scala: https://docs.scala-lang.org/tour/classes.html
- Tipos como objetos: https://docs.scala-lang.org/tour/basics.html#scala-basics

Ejemplo:

```scala
trait Imprimible {
  def imprimir(): Unit
}

class Persona(val nombre: String) extends Imprimible {
  def imprimir(): Unit = println(s"Persona: $nombre")
}
```

### Sistema de tipos y chequeos de tipos
Scala posee un sistema de tipos estático y fuerte con inferencia extensa.

- Tipos estáticos verificados en compilación.
- Inferencia de tipos reduce la necesidad de anotaciones explícitas.
- Genéricos, `trait`, `case class` y `type` para modelar dominios.
- `Option` y algebra de tipos ayudan a evitar valores nulos.

Fuentes:
- Tipos estáticos y inferencia: https://docs.scala-lang.org/tour/basics.html#scala-basics
- Option: https://www.scala-lang.org/api/current/scala/Option.html
- Case classes: https://docs.scala-lang.org/tour/case-classes.html
- Pattern matching: https://docs.scala-lang.org/tour/pattern-matching.html

Scala realiza la mayoría de los chequeos de tipos en tiempo de compilación; la ejecución en la JVM conserva la semántica definida por el compilador.

Ejemplo:

```scala
def resumen(tokens: List[String]): Int = tokens.length

val palabras = List("enero", "febrero")
val largo = resumen(palabras)
```

Este análisis muestra que Scala ofrece un equilibrio entre estructuras orientadas a objetos y mecanismos funcionales, con un sistema de tipos estático capaz de detectar errores antes de ejecutar el programa.
